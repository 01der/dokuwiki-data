===== Database Administration Tools with Support of SQL Databases of a single Vendor =====
==== Microsoft SQL Server Administration ====
  *The free [[http://www.db3nf.com/|DB3NF]] - "Rapid Application Development platform (RAD for Web)", for use with Microsoft SQL Server and Microsoft IIS.
  *The free fabFORCE DBDesigner 4 & free follow-ups.
    *[[http://www.fabforce.net/dbdesigner4/|fabFORCE DBDesigner 4]] for Windows, Linux. Kylix 3 / Delphi 7 sources.
      *[[http://en.wikipedia.org/wiki/DBDesigner_4|EN.Wikipedia "DBDesigner 4"]], [[http://de.wikipedia.org/wiki/DBDesigner_4|DE.Wikipedia "DBDesigner 4"]].
  *The free [[http://www.devart.com/dbforge/sql/sqlcomplete/|devart "dbForge SQL Complete"]] for add-in for SQL Server Management Studio Professional (2012, 2008) and Visual Studio Profesional - "SQL code formatter offer powerful autocompletion and formatting of T-SQL code that replaces native Microsoft T-SQL Intellisense".
  *Microsoft SQL Server Data Tools ( SSDT ).
    *[[http://msdn.microsoft.com/en-us/data/tools.aspx|Microsoft Data Developer Center "Microsoft SQL Server Data Tools .."]].
    *[[http://blogs.msdn.com/b/ssdt/|SQL Server Data Tools Team Blog]] - "Offical team blog for SSDT, a tool for on and off-premise database development".
      *[[http://blogs.msdn.com/b/ssdt/archive/2014/01/31/ssdt-and-visual-studio-versions.aspx?utm_source=feedburner&utm_medium=feed&utm_campaign=Feed%3A+ssdtblog+%28SQL+Server+Developer+Tools+Team+Blog%29|SQL Server Data Tools Team Blog "SSDT and Visual Studio versions"]].
    *"Microsoft Visual Studio 2013 for Windows Desktop" is shipped with SSDT, -> see [[sdevtools.html|Software Development Tools]].
      *You may do SQL unit tests with "Microsoft SQL Server Data Tools ( SSDT )" of "Microsoft Visual Studio Professional" and "Microsoft Visual Studio Ultimate".
      *The free "Microsoft Visual Studio Express 2013" is shipped with an edition of "Microsoft SQL Server Data Tools ( SSDT )" which lacks the unit test feature.
      *All editions of "Microsoft Visual Studio 2013" are shipped with a "LocalDB" of SQL Server Express. So for development, you are not urged anymore to install "SQL Server Express" ( as a Windows service.. ) anymore on your development machines.
  *Tools shipped with Microsoft SQL Server:
    *[[http://msdn.microsoft.com/en-us/library/ms162802.aspx|Microsoft MSDN "bcp Utility"]] - "The bcp utility bulk copies data between an instance of Microsoft SQL Server and a data file in a user-specified format. The bcp utility can be used to import large numbers of new rows into SQL Server tables or to export data out of tables into data files".

  -The free [[http://www.saxmike.com/MySoftware/MySoftware.asp?Menu=MYSOFTWARE|Mike T's SQLite Database Administrator Tool]] for Windows, Linux, MaxOSX.
    *You may open several query windows concurrently.
    *You may not load a query to a file, due to a software failure ( a loaded query is parsed, resulting in an empty query window ) :-(. Instead, you must use copy+paste.
    *You may save and work with several query windows, concurrently :-).
    *You may not include comments ( /* comment */ ) in your query.
  -The OpenSource [[http://github.com/slashmili/PySQLiteGUI|Github "slashmili / PySQLiteGUI"]] in Python.

    *You may not load a query from or save a query to a file :-(. Instead, you must use copy+paste.
  -The free [[http://sqliteadmin.orbmu2k.de/|orbmu2k "SQLite Administrator"]] for Windows. 
    *You may load, save and work with several query windows, concurrently :-). But though You may save queries created within "SQLite Administrator" to files ( by "Save To File" ), there is no "Load From File" option in the query menu :-(. So You may load contents to query windows just by copy+paste.
    *You may not rename saved queries:-(.
    *Unexpected & unwanted behaviour of the user interface: CTRL-S does not save anything, but deletes the current query window, including history :-(.
    *Trigger operations ( Create, Edit, Delete ) :-).
  -The OpenSource [[http://sqlitebrowser.sourceforge.net/|SourceForge "SQLite Database Browser"]], [[http://www.sourceforge.net/projects/sqlitebrowser/|SourceForge "SQLite Database Browser"]] for Windows with Qt.
    *There is just one single query window :-(.
    *You may not load a query from or save a query to a file :-(. Instead, you must use copy+paste.
  -The OpenSource [[http://sqlitedesigner.sourceforge.net/|SourceForge "SQlite Designer"]], [[http://www.sourceforge.net/projects/sqlitedesigner/|SourceForge "SQlite Designer"]] for WinXP.
  -The free [[http://www.sqliteexpert.com/|SQLite Expert Personal]] for Windows #.
    *[[http://www.sqliteexpert.com/features.html|SQLite Expert - Feature comparison]].
    *You may load, save and work with several query windows, concurrently :-).
    *Trigger operations ( Edit Trigger Table ) :-).
    *A query window is not linked with a loaded file. I.e. there is no "Save" function ( expected/ususal keyboard shortcut "CTRL-S" ), but just "Save as" :-(.
    *In case of unsuccessful queries, the reason for failure is explained in a modal window, but just after an extra click on a "Details" button. However, the native SQLite error message is not displayed.
    *[[http://www.sqliteexpert.com/tutorials.html|SQLite Expert - Flash tutorials]].
  -The free [[http://www.singular.gr/sqlite/|Mike Cariotoglou "Sqlite3Explorer"]] for Windows.
    *You may load, save and work with several query windows, concurrently :-).
  -The OpenSource [[http://www.sqliteman.com/|Sqliteman - Sqlite Databases Made Easy]], [[http://sqliteman.sourceforge.net/|SourceForge "Sqliteman. Sqlite Databases Made Easy"]], [[http://www.sourceforge.net/projects/sqliteman/|SourceForge "sqliteman: sqlite3 admin and devel tool"]] for Windows and Linux.

  -The OpenSource browser add-on [[http://code.google.com/p/sqlite-manager/|Google Code "sqlite-manager. Extension for Firefox and other apps to manage any sqlite database"]], [[http://addons.mozilla.org/en-US/firefox/addon/sqlite-manager/|Mozilla Firefox Add-Ons "SQLite Manager"]].

    *In case of unsuccessful queries, the reason for failure is explained in a modal window. However, the native SQLite error message is not displayed :-(.
  -The DonationWare [[http://www.sqlitestudio.pl/|SQLite Studio]] for Windows.
    *You may load and save queries :-), in one single query window :-(.
  -The OpenSource [[http://sqlitemanager.codeplex.com/|CodePlex "SQLite Studio Management"]] for Windows with .NET 2.0.
